Assim que criamos uma nova pasta ou arquivo a IDE, no meu caso o VS Code fica um U (Untracked) ao lado do nome do arquivo, o que nos indica que as alterações não estão sendo monitoradas, ou seja, o github desktop não consegue fazer uma comparacao do conteudo da IDE com o do repositorio remoto, afinal esse documento não existe para que possa fazer essa comparação e sugerir uma commit


Ao fazer o commit temos outro tipo de aviso o M


Ja o M simboliza que aquele documento que o github desktop acoplado ao VSCODE consegue monitorar suas alterações na IDE e sugerir commits para salvá-las no repositorio local foi modificado. O conteudo que está lá no repositorio local já não é o mesmo que está na IDE, ou seja ele ja sabe que o programador digitou coisa nova. Dai o aviso M (modificado).


Ou seja, após o primeiro commit após a criação de um novo documento já passamso a ter um paramentro de comparação para a sugestão da realização de commits
